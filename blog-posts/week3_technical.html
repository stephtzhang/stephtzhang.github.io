JavaScript: Pros and Cons

  • BLOG
  • PROJS

JavaScript was first released under the name Mocha in 1995. Its original creators intended it to appeal to non-professional programmers with its relative simplicity. Nowadays, JavaScript remains popular because of that simplicity. It also loads quickly because it is a client-side, rather than server-side, language. This means JavaScript code loads on a client's local machine, as opposed to a remote server. JavaScript libraries like jQuery and platforms built on JavaScript like node.js further increase the appeal of the language.

Despite the upside of using JavaScript, its client-side execution creates a few problems. Firstly, JavaScript creates security issues because it can potentially insert malicious code into an html element or open an unsafe url. Secondly, JavaScript may be interepreted differently by different browsers on the client's machine, causing variations.
