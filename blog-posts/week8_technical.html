Web security risk grows relative to the number of users who access a site or application and the value of the information that passes through it (ie financial or identity information). For that reason, many young applications do not prioritize security. The chief marketing officer at Webroot describes, "There is a threshold there where the malware writers feel like they can monetise and make some good return on investment to go after that group of users, and that is the point where we have to go look at it seriously."

Controlling user input and using a web scanner are two fundamental ways to protect a site or application.

Any interaction with users that accepts input opens a site to vulnerabilities. To prevent users from inserting malicious code into an application, you can call the HtmlEncode method. HtmlEncode renders user-supplied information as static text, so that any malicious code that is inputted will not be interpreted.

In situations where you would like user input to trigger an action, use input to set parameter values rather than directly accepting potentially malicious code.

While an ideal world would allow limitless resources for maintaining a site or application's security, regularly using one of many available web scanners to test your code is a very effective starting point. These scanners find weaknesses by performing common attacks.

Web scanners cannot protect a site or application against newly developped hacking strategies, however known hacking techniques are far more likely to occur than unknown ones because of the cost and time necessary to find new vulnerabilities.
