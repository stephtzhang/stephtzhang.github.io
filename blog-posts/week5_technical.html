Hashes for Init Args

  • ABOUT
  • BLOG
  • PROJS

Using hashes for initialization arguments is an efficient way to decrease dependency between objects and make code more flexible for unpredicted future changes. If we take a look at this week's two die assignments, we have two similar classes that take different input into their initialization method.

The first die class took a positive integer that represented the number of sides on a die as its sole argument:

def initialize(sides)

@sides = sides

end

The second die class took an array representing the values on all the sides of a die as its sole argument:

def initialize(labels)

@labels = labels

end

Using a hash for an initialization argument for a single die class, cuts down on code and makes the class more flexible for future change:

def initialize(args)

@sides = args[:sides]

@labels = args[:labels] || (1..@sides).to_a

end

Calling either Die.new(sides: some_int) or Die.new(labels: some_arr) will initialize a die object with the above initialization method. The @labels variable will be passed the value at :label from the args hash if it exists. If not, it will be set to an array of numbers based on the value at :sides from the args hash.

In Practical Object-Oriented Design in Ruby, Sandi Metz encourages the use of hashes for initialization methods because it reduces dependency on argument order, makes more stable code, and provides documentation about the arguments.
